# Enterprise Automation Core: System Overview

## 1. Executive Summary
This document specifies the software-defined automation architecture for the uncrewed operation of the Seattle Center Monorail (Alweg Mark II, 1961 trainsets). The core logic merges legacy electromechanical drivetrains with Cold War tactical computing frameworks and modern, cloud-based fleet telemetry.

## 2. Multi-Generational Architecture
The control platform decouples real-time high-speed navigation from localized physical actuation across a five-tier structural stack:

*   **Tier 1: Cloud Intelligence & Safety Matrix** — Managed via Samsara AG/VG dual-gateway arrays tracking point-by-point route profiling, weather indexes, and historical deceleration boundaries.
*   **Tier 2: Real-Time Edge Processing** — A Linux-based open PLC/Industrial computer execution layer running containerized Python tracking engines.
*   **Tier 3: Telemechanical Signal Marshalling** — Interfaced to a 1957 UNIVAC Model 17 and Typewriter interface terminal executing 32-bit parallel calculations.
*   **Tier 4: Mechanical Guidance & Attitude Control** — Driven by a sealed Chrysler Aerospace guidance node monitoring chassis pitch, roll, and lateral G-forces.
*   **Tier 5: Locomotion & Pneumatic Braking** — Direct interface lines to 1961 General Electric 700V DC traction motors and the Westinghouse air brake pipe network.
